JavaScript面向对象-继承

发表于 2016-02-03 | 分类于 JavaScript |

JavaScript不同于其他面向对象的类式继承模型。而是提供了原型继承模型prototype，这一方法往往被人当成缺陷诟病。当使用原型继承有时会比类式继承更加强大，但是使用往往也要复杂的多。

类式继承

类式继承原理

这里需要注意在类式继承中将第一个值的实例赋值给第二个类的原型。

SuperClass这个类中分别包含构造函数内的属性方法和原型中的属性方法。
类的原型对象作用是为为类的原型添加共有方法，而类本事不能直接访问这些属性和方法，需要通过class.prototype的形式访问。而我们实例化一个父类是，新创建的对象复制了父类构造函数的属性与方法，并且将__proto__指向了父类的原型对象，这样就拥有了父类原型对象的属性和方法。

类式继承缺陷

  • 由于子类通过自身原型prototype对父类实例化，继承了父类。如果父类中共有属性为引用类型，就会在子类中被所有实例共用。因此一个子类实例更改从父类构造函数中继承的共有属性，就会影响其他子类。示例如下：

  • 由于子类实现的继承是通过自己prototype对父类的实例化实现，因此在创建父类时，是无法向父类传递参数的，因此在父类实例化时也无法对父类构造函数内属性进行初始化。

构造函数继承

构造函数继承原理

SuperClass.call(this,id)这里通过call方法来更改函数的作用环境，因此在子类中，对SuperClass调用call方法就是将子类中的变量在父类中执行一遍，由于父类中给this绑定了属性，因此子类就继承了父类中的共有属性。

构造函数继承缺陷

构造函数继承本身并没有涉及到prototype，所以父类的原型方法并不会被子类所继承。而将方法放到构造函数中，则被子类所实例化的每一个对象都会单独拥有一份且不能共有，这就违反了代码复用的原则，于是有了第三种方式–组合继承。

寄生式继承

寄生式继承原理

寄生式继承就是对原型式继承的第二次封装，并且在此之上对继承的对象进行拓展,使其可以添加新的属性和方法。

寄生式继承缺陷

同原型式继承与类式继承一样，改变从父类继承的引用类型得知会使每一个实例化的对象的值都被改变。
